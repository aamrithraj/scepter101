Sale Confirmation Force Date -Sale Confirm Backdate Odoo App

Sales Order Confirmation Back Date and Force Date Odoo Apps

Sales Confirmation Force Date odoo app helps users to select sale confirmation date forcefully while confirming the sales order and selected date set as confirmation date on sales order. This selected date also reflects the delivery order as the schedule date, same as default Odoo works. If the user set delivery date on sales order then confirmation date only applies to the sales order and delivery date set as schedule date on delivery order.

Features

Force Select Sale Confirmation Date

User can force select confirmation date for sale order.

Reflect Force Confirmation Date on Delivery Order

Selected force sale confirmation date also reflected on delivery order only if delivery date not selected.

Sale Confirmation Force Date Wizard

On clicking "CONFIRM SALE" button new wizard will open.

On "Select Confirmation Force Date" wizard user have to select date that he/she wants to set as sale confirmation date.

User can see selected date is set as order date for sale order.

User can also see that selected confirmation date is also set as "Scheduled Date" for delivery order.

Sale Confirmation Force Date with default Delivery Date

User can see created request for quotation with current date, also "Delivery Date" is selected.

User can see selected sale confirmation date.

User can see selected date is set as sale confirmation date.

Here user can see that delivery date is same as selected before on sale order and not changed.
